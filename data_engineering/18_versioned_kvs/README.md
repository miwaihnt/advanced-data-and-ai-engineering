# 💻 課題18：バージョン付きキーバリューストア（Versioned Key-Value Store）

本プロジェクトは、エンタープライズ向けのデータプラットフォームにおけるデータのタイムトラベルや状態の履歴管理を可能にする「バージョン付きキーバリューストア」の設計・実装検証です。
顧客の要件として、「ある設定値が、過去のある時点でどういう値だったかを遡って参照できるバージョン管理付きのデータストア」が必要になりました。

例えば、機械学習モデルのハイパーパラメータや、A/Bテストの設定値を時系列で記録し、「2時間前の時点ではどの設定が適用されていたか？」を高速に引ける仕組みを作ります。

以下の2つの操作をサポートする `TimeMap` クラスを実装してください。

---

### 📌 満たすべきインターフェースと要件：

#### `set(key: str, value: str, timestamp: int) -> None`
- 指定された `key` に対して、`timestamp` 時点での `value` を記録する。
- 同じ `key` に対して、異なるタイムスタンプで複数回呼ばれることがある（履歴が蓄積される）。

#### `get(key: str, timestamp: int) -> str`
- 指定された `key` に対して、`timestamp` **以前（その時刻を含む）** に記録された値のうち、最も新しいものを返す。
- 該当する値が存在しない場合は、空文字 `""` を返す。

---

### 📌 制約と要件（シニアの防衛策）：

1. **計算量の最適化**:
   - `set` は O(1) で動作すること。
   - `get` は O(log N) で動作すること（N はそのキーに紐づく履歴の数）。全履歴を線形探索（O(N)）するのは不合格とします。
2. **データ構造の選定**:
   - Python標準ライブラリの `bisect` モジュール（二分探索）を活用して、ソート済みリストから効率的に値を引く設計にしなさい。
3. **堅牢なエッジケース処理**:
   - 存在しない `key` を `get` しても `KeyError` で落ちないこと。
   - 指定した `timestamp` より前にデータが一切ない場合は、空文字 `""` を返すこと。

---

### 📥 入力と期待される動作のイメージ

```python
tm = TimeMap()
tm.set("config_lr", "0.01", 100)
tm.set("config_lr", "0.001", 200)
tm.set("config_lr", "0.0001", 300)

tm.get("config_lr", 150)  # -> "0.01"   (timestamp 100 で設定された値)
tm.get("config_lr", 200)  # -> "0.001"  (timestamp 200 ちょうどに設定された値)
tm.get("config_lr", 350)  # -> "0.0001" (timestamp 300 で設定された値が最新)
tm.get("config_lr", 50)   # -> ""       (timestamp 100 より前にはデータなし)
tm.get("unknown_key", 100) # -> ""      (存在しないキー)
```

---

### 💡 本実装パターンの重要性と実務上の価値

- **二分探索の実践的応用**: LeetCodeで学ぶ二分探索を、「時系列データの履歴検索」という実務シナリオに落とし込んで使えるかを見られます。
- **複合データ構造の設計力**: 「辞書（ハッシュマップ）の中にソート済みリストを入れる」という複合的なデータ構造を自分で設計・実装できるかが試されます。
- **アーキテクチャ的な視点**: 顧客が「過去のある時点の状態を知りたい」という要件を、コードレベルでどう解決するかの設計力。
