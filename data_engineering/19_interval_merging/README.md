
課題19：重複する時間区間のマージ（Interval Merging）

### 【ユースケースとシステム背景】
あんたはデータ基盤チームのシニアエンジニアです。
バッチジョブのスケジューラから、各ジョブの「実行時間帯（開始時刻, 終了時刻）」のリストが渡されます。
複数のジョブの実行時間帯が重なっている場合は、それらを1つの連続した時間帯にマージ（統合）し、「実際にシステムが稼働していた時間帯」の一覧を求めてください。

これは、以下のような実務シナリオで頻出するパターンです：
- CloudWatch / Datadog のアラート期間のマージ（「実際にシステムが不安定だった合計時間」の算出）
- ETLジョブのスケジュール最適化（重複するバッチ実行ウィンドウの統合）
- ログ解析における「ユーザーのアクティブ時間帯」の集約

---

### 📌 要件：

#### `merge_intervals(intervals: List[Tuple[int, int]]) -> List[Tuple[int, int]]`

- 入力: `(start, end)` のタプルからなるリスト。ソートされているとは限らない。
- 出力: 重複する区間をすべてマージした後の、開始時刻の昇順にソートされた区間リスト。

---

### 📌 制約と要件（シニアの防衛策）：

1. **計算量の最適化**:
   - 時間計算量 O(N log N) で解くこと（ソート + 線形スキャン）。
   - O(N^2) の総当たり比較は不合格とします。
2. **エッジケースの堅牢な処理**:
   - 空リスト `[]` が渡された場合は、空リスト `[]` を返すこと。
   - 区間が1つしかない場合も正しく動作すること。
   - 完全に包含される区間（例: `(1, 10)` の中に `(3, 5)` が入っている場合）を正しく処理すること。
   - 隣接する区間（例: `(1, 3)` と `(3, 5)`）はマージ対象とすること。
3. **入力を非破壊に処理すること**:
   - 元のリストをソートで書き換えず、コピーを作ってから処理するのが望ましい。

---

### 📥 入力と期待される出力

```python
# テストケース1: 正常系（重複あり）
intervals_1 = [(1, 3), (2, 6), (8, 10), (15, 18)]
# 期待: [(1, 6), (8, 10), (15, 18)]

# テストケース2: 完全包含
intervals_2 = [(1, 10), (3, 5), (4, 7)]
# 期待: [(1, 10)]

# テストケース3: 隣接（境界が接する）
intervals_3 = [(1, 3), (3, 5), (5, 7)]
# 期待: [(1, 7)]

# テストケース4: 重複なし
intervals_4 = [(1, 2), (5, 6), (9, 10)]
# 期待: [(1, 2), (5, 6), (9, 10)]

# テストケース5: 空リスト
intervals_5 = []
# 期待: []
```

---

### 💡 アルゴリズムのヒント: 「Sort + Sweep（ソート＋スイープライン）」パターン

1. まず区間を**開始時刻（start）でソート**する。
2. ソート済みの区間を左から順にスキャン（sweep）していく。
3. 現在の区間の start が、直前にマージした区間の end 以下であれば、重複しているのでマージする（end を max で更新）。
4. そうでなければ、新しい独立した区間として結果リストに追加する。

### 💡 本実装パターンの重要性と実務上の価値

- **ソート済みデータに対する線形スキャン**: ソートで O(N log N) + スキャンで O(N) = 全体 O(N log N) という、DE実務のゴールデンパターン。
- **境界条件のハンドリング**: 「完全包含」「隣接」「重複なし」というエッジケースの網羅的な処理は、プロダクションコード品質の指標。
- **データパイプラインへの直結**: ETLジョブのウィンドウ最適化やアラート期間の統合など、DE実務で即活用できる。
