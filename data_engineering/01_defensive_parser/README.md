## 課題1：異常値でクラッシュさせない「防御的パースとクレンジング」

APIから取得したセンサーデータ（JSON文字列のリスト）を辞書型にパースし、正常なデータのみを抽出してください。

### 要件:
- 文字列をJSONとしてパースする。パースに失敗した（破損したJSON）行はスキップする。
- sensor_id と temperature キーが必須。欠損している行はスキップ。
- temperature は数値型（int または float）でなければならない。文字列で入っている場合は数値にキャストする。キャストできない値（例: "error"）の場合はスキップ。

- 正常に処理できた辞書のリストを返すこと。

入力例:

```
raw_data = [
'{"sensor_id": "s1", "temperature": 22.5}',
'{"sensor_id": "s2"}', # temperature欠損
'{"sensor_id": "s3", "temperature": "25.0"}', # 文字列の数値
'{"sensor_id": "s4", "temperature": "NaN"}', # パース不可な文字列
'invalid_json_string', # JSON破損
'{"sensor_id": "s5", "temperature": 18}'
]

```
*   **期待される出力:**
    ```python
    [
        {"sensor_id": "s1", "temperature": 22.5},
        {"sensor_id": "s3", "temperature": 25.0},
        {"sensor_id": "s5", "temperature": 18.0}
    ]

## エラーの使い分け
  1. なぜ ValueError と TypeError なのか
   * ValueError: 型は合ってる（文字列とか）けど、中身が数値として解釈できないときに出るわ。（例: float("abc")）
   * TypeError: そもそも float() に渡しちゃいけない型が来たときに出るわ。（例: float(None) や float([1, 2])）
  だから、数値変換のガードとしてはその2つで正解よ。

  2. あんたのコード、まだ穴があるわよ！
  あんたの input.jsonl にこれがあったわよね？
  {"sensor_id": "s4", "temperature": "NaN"}

  実はね、float("NaN") や float("inf") はエラーにならずに通過しちゃうのよ！
  Pythonの float 型としては正しい値だから、ValueError は出ないわ。でも、温度データとして「非数（NaN）」や「無限大（inf）」が紛れ込んでもいいのかしら？


## 数値への変換
✦ あんた、本当に数字の扱いに疎いわね！そんなんじゃ計算もろくにできないバカ犬になっちゃうわよ！

  いい？Pythonの int() 関数はね、「純粋な数字の文字列」しか直接受け付けないのよ。

  なぜ int("25.0") はエラーになるの？
   1. float("25.0") の場合:
     Pythonは「あ、これ小数点のついた数字ね」と理解して、ちゃんと 25.0 という数値に変換してくれるわ。
   2. int("25.0") の場合:
     Pythonは「はぁ！？整数（int）にするって言ったのに、なんでドット（.）なんて混ざってるのよ！そんなの整数じゃないわよ！」って怒って ValueError を投げるの。

  つまり、文字列の中にドットが含まれている時点で、int() は「自分には扱えないフォーマットだわ」って匙を投げるのよ。一度 float にしてから int
  にすれば通るけど、いきなり int は無理ってこと！

  なぜ int("NaN") はエラーになるの？
  これはもっと当たり前でしょ！ NaN（Not a Number）は「数値ではないもの」を表す特殊な浮動小数点数値なの。
  float 型には NaN という概念があるけど、整数（int）の世界には「数値じゃない整数」なんて存在しないわ。
  だから、int() に NaN なんて渡したら「ふざけないで！」ってエラーになるに決まってるじゃない！

  整理するとこうよ！
  ┌────────────┬────────────────┬─────────────────────┐
  │ 入力文字列 │ float() の結果 │ int() の結果        │
  ├────────────┼────────────────┼─────────────────────┤
  │ "25"       │ 25.0 (成功)    │ 25 (成功)           │
  │ "25.0"     │ 25.0 (成功)    │ ValueError (失敗！) │
  │ "NaN"      │ nan (成功)     │ ValueError (失敗！) │
  └────────────┴────────────────┴─────────────────────┘

  あんたがもし「どんな形式の数字でも整数にしたい」って思うなら、まず float() で数値化してから、その後に int()
  で端数を切り捨てるか、四捨五入するかしないとダメなのよ。

  わかった？数値の型にはそれぞれ「許せる範囲」があるんだから、勝手に自分の都合を押し付けないことね！ふんっ！
