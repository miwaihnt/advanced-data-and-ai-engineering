💻 課題10：リアルタイム移動窓集計（Sliding Window with Deque）
【頻出企業】 AWS (CloudWatchチーム), Meta, Databricks (ストリーミング部隊)

【ビジネス背景・お題】
ストリーミング基盤において、リアルタイムの異常検知（Anomaly Detection）を行うコンポーネントを設計しています。系统的監視エージェントから、エラーが発生するたびにその timestamp（秒単位の整数または浮動小数点数）がリアルタイムで関数に流れてきます。

あんたの任務は、新しいエラーログが到着するたびに、「今この瞬間（到着した秒）から、過去60秒間（[timestamp - 60, timestamp]）の間に何件のエラーが発生したか」を即座に（$O(1)$に近い速度で）計算して返す監視クラス ErrorMonitor を実装することです。

📌 制約と要件：
メモリを使い潰さないこと： システムが1年間稼働したからといって、1年分のタイムスタンプを配列に保持し続けるのは OOM（メモリ枯渇）で即死 するため絶対NGです。

窓外データの即時破棄： 60秒の窓（Window）から外れた古いデータは、新しいデータが到着したタイミングで速やかにメモリから破棄（削除）してください。

データ構造の選定： collections.deque（両端キュー）の性質を活かすのがシニアの最適解です（タイムスタンプは常に単調増加、つまり古い順に流れてくる前提で構いません）。
