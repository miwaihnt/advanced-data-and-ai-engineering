# 課題15：Polarsによる大容量データの遅延評価（LazyFrame）と窓関数集計（High-Performance Polars Pipeline）

### 【ユースケースとシステム背景】
あなたは大規模なアドテック企業のデータエンジニアです。
配信された広告に対するユーザーの「クリックイベントログ（数千万行〜数億行のCSV形式）」を処理し、リアルタイムでの配信最適化モデルのために「ユーザーごとの過去5分間の移動平均クリック単価（CPC）」を計算するバッチ集計パイプラインを設計しています。

この処理を Python の素のループや標準の Pandas（`read_csv`）で行うと、数GBものデータをすべてメモリに載せるため、数分で OOM（メモリ枯渇）を引き起こしてしまいます。

極めて高速かつメモリ効率の高いライブラリである **Polars** の **LazyFrame API** を駆使し、
- ファイルをメモリにロードせずにクエリ式のみを組み立てる（遅延評価）
- メモリ制限を超えるデータをディスクと連携させながらストリーミング処理する（OutOfCore）
手法を実装しなさい。

---

### 🚨 設計において考慮すべき極限状態の要件：

1. **インメモリ展開の完全回避（Lazy API の使用）:**
   データファイルを即座に読み込む `pl.read_csv` は使用禁止です。メタデータのみを高速に読み込み、クエリグラフをビルドする **`pl.scan_csv`** を使用しなさい。

2. **データ型キャストとクレンジング:**
   - タイムスタンプ（Unixエポック秒：整数）を Polars の Datetime 型に変換しなさい（`pl.from_epoch` を使用）。
   - クリーニングとして、無効なレコード（クリック額がマイナス、または特定の必須列がNULLのもの）をフィルタリングしなさい。

3. **移動窓（Rolling Window）を用いた集計の設計:**
   - ユーザーID（`user_id`）ごとにグループ化し、イベント時刻（`timestamp`）の順にソートした上で、**「過去5分間（`5m`）の移動平均クリック金額（`amount`）」**を算出しなさい。
   - Polars の超強力なストリーミング集計メソッドである **`rolling`** を活用しなさい。

4. **ストリーミングライト（OutOfCore Sink）:**
   集計結果をメモリ上に巨大なDataFrameとしてロードせず、ディスクへ直接ストリーミング保存する **`sink_parquet()`** を使用し、最初から最後まで極小のメモリフットプリントで完走させなさい。

---

### 📥 入力データの仕様

入力は CSV 形式のファイルです。
```csv
timestamp,user_id,amount,status
1719327600,u1,120.5,success
1719327615,u1,80.0,success
1719327630,u2,300.0,success
...
```
※ timestampはUnixエポック秒を表す整数。

### 💡 この課題で学べること（シニアDEの必須スキル）

- **Eager API と Lazy API の決定的な違い:** すぐに実行するのではなく、最適化（フィルタプッシュダウン、列射影など）された論理計画（Query Plan）を組み立ててから実行する重要性。
- **OutOfCore（ストリーミング）実行:** メモリのサイズを超えるテラバイト級のデータを処理するためのディスクバッファと並列I/Oパイプライン。
- **時系列エンジニアリング:** `rolling` や `group_by_dynamic` を用いた、タイムスタンプベースの高度なスライディングウィンドウ集計。
- **Parquetの優位性:** 分析処理における列指向バイナリ形式の優位性と、Polarsからの直接シリアライズ。
