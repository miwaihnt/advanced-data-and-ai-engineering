# 課題31：PySpark 構造化ストリーミング with ウォーターマーク（遅延データ対応）

## 🎯 課題概要

本番のストリーミングパイプラインでは、ネットワーク遅延やシステム障害により、**イベント発生時刻（event_time）よりも遅れてデータが届く「遅延データ（Late Data）」**が必ず発生する。

本課題では、PySpark Structured Streaming の `withWatermark` と `window` 関数を使い、**イベント時刻ベースの時間窓集計**と**遅延データの自動棄却**を実装する。

**使用言語**: PySpark (Structured Streaming API)

---

## 📋 前提シナリオ

Webサービスのアクセスログが、Kafkaに似たソケットストリームとして届く。
各イベントは `event_time`（実際の発生時刻）と `user_id`、`page` を持つ。

```
{event_time: "2024-01-01 10:00:00", user_id: "U-1", page: "/home"}
{event_time: "2024-01-01 10:01:30", user_id: "U-2", page: "/product"}
{event_time: "2024-01-01 09:55:00", user_id: "U-3", page: "/home"}  ← 5分遅延
...
```

---

## ✅ 実装要件

### Part 1：ウォーターマーク付き時間窓集計

- `event_time` カラムを `timestamp` 型にキャストすること。
- `withWatermark("event_time", "10 minutes")` を設定すること。
  - これは「最大10分遅れのデータまで受け入れ、それ以上遅延したデータは無視する」ことを意味する。
- `window("event_time", "5 minutes")` のタンブリングウィンドウで集計すること。
- 各5分ウィンドウ内の `page` ごとのアクセス数（`count`）を集計すること。

### Part 2：rate ソースを使ったローカル検証

- テスト用に `spark.readStream.format("rate")` を使い、ローカルで動作確認できる
  シンプルなストリームを作成すること（ソケット接続は不要）。
- `outputMode("append")` と `format("console")` でコンソール出力すること。
